## Documentation Tools & Standards

### Writing Standards
- **Format:** Markdown (.md)
- **Line Length:** Max 120 characters
- **Headers:** ATX style (#, ##, ###)
- **Lists:** Dash (-) for unordered
- **Code Blocks:** Triple backticks with language
- **Links:** Relative paths when possible
- **Dates:** ISO format (YYYY-MM-DD) or spelled out

### Organization Standards
- **Naming:** UPPERCASE_WITH_UNDERSCORES for audit files
- **Location:** Feature docs in feature folders
- **History:** Keep old state docs for reference
